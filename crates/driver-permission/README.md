# nexo-driver-permission

> Phase 67.3 — MCP `permission_prompt` tool that gates Claude Code tool calls. When Claude is launched with `--permission-prompt-tool mcp__nexo-driver__permission_prompt`, every proposed `Edit`/`Bash`/`Write`/… flows through here before execution.

This crate is part of **[Nexo](https://github.com/lordmacu/nexo-rs)** — a multi-agent Rust framework with a NATS event bus, pluggable LLM providers (MiniMax, Anthropic, OpenAI-compat, Gemini, DeepSeek), per-agent credentials, MCP support, and channel plugins for WhatsApp, Telegram, Email, and Browser (CDP).

- **Main repo:** <https://github.com/lordmacu/nexo-rs>
- **Runtime engine:** [`nexo-core`](https://github.com/lordmacu/nexo-rs/tree/main/crates/core)
- **Public docs:** <https://lordmacu.github.io/nexo-rs/>

## Status

**Internal crate — not published to crates.io.** Marked
`publish = false, release = false` in `release-plz.toml`.

## What this crate ships

- **Types** (`PermissionRequest`, `PermissionOutcome`, `PermissionResponse`):
  serde-tagged, NATS-safe.
- **Trait** `PermissionDecider`: the side that actually decides.
  AllowAll / DenyAll / Scripted impls live here for tests.
- **MCP server**: `PermissionMcpServer<D>` impl `McpServerHandler`
  with one tool `permission_prompt`.
- **Bin**: `nexo-driver-permission-mcp` runs the MCP server over stdio
  so Claude Code can spawn it as a child.

## What this crate does NOT ship (yet)

- Wiring the bin to the daemon — Phase 67.4 will add a
  `--socket <path>` flag and an `LlmDecider` consulting MiniMax.
- The `mcp-config.json` Claude Code reads to discover this server —
  generated by the driver loop (67.4) on goal start.
- Telemetry counters — Phase 67.6.

## Bin modes (67.3 placeholder)

- No flags → `AllowAllDecider` with a loud `tracing::warn!`. **Dev only.**
- `--deny-all <reason>` → `DenyAllDecider`. Useful for shadow-mode
  tests (Phase 67.11).

Both go away when 67.4 ships `--socket`.
