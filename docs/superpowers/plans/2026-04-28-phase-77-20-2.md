# Phase 77.20.2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Intercept `Sleep` at driver-loop runtime, park the in-process goal, and resume it with a synthetic `<tick>` prompt instead of replaying the Sleep sentinel to the model.

**Architecture:** Add a typed `AttemptOutcome::Sleep` seam in `crates/driver-types`, teach `attempt.rs` to translate the sentinel into that outcome, and isolate sleep/wake helpers in a new `crates/driver-loop/src/proactive.rs` module so `orchestrator.rs` can schedule a cancellation-aware wake-up without overloading replay-policy paths.

**Tech Stack:** Rust, Tokio, serde/serde_json, existing `driver-loop` integration test harness, `nexo_core::agent::sleep_tool` sentinel helpers.

- [ ] **Step 4: Implement the proactive helper module**

Create `crates/driver-loop/src/proactive.rs` with the runtime-local wake representation and cancellation-aware wait:

```rust
use std::time::{Duration, Instant};

use tokio::time::sleep;
use tokio_util::sync::CancellationToken;

#[derive(Clone, Debug)]
pub struct ScheduledWake {
    pub duration_ms: u64,
    pub reason: String,
    pub sleep_started_at: Instant,
}

#[derive(Clone, Copy, Debug, PartialEq, Eq)]
pub enum WakeResult {
    Fired { elapsed_ms: u64 },
    Cancelled,
}

pub fn build_tick_prompt(wake: &ScheduledWake, elapsed_ms: u64) -> String {
    format!(
        "<tick>\nkind: sleep_wake\nelapsed_ms: {elapsed_ms}\nreason: {}\n</tick>",
        wake.reason
    )
}

pub async fn wait_for_wake(wake: &ScheduledWake, cancel: &CancellationToken) -> WakeResult {
    let duration = Duration::from_millis(wake.duration_ms);
    tokio::select! {
        _ = sleep(duration) => WakeResult::Fired {
            elapsed_ms: wake.sleep_started_at.elapsed().as_millis() as u64,
        },
        _ = cancel.cancelled() => WakeResult::Cancelled,
    }
}
```

use crate::proactive::{build_tick_prompt, wait_for_wake, ScheduledWake, WakeResult};

AttemptOutcome::Sleep { duration_ms, reason } => {
