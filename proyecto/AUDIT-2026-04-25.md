## 🔴 Critical — fix before next ship

### C1. Panics in transcript JSONL parsing
- `crates/core/src/agent/transcripts.rs:248,255,351`
- `panic!("first line must be a Session header")` and friends fire
  on malformed JSONL. A corrupted transcript file (truncated write,
  manual edit, partial flush) crashes the agent.
- **Fix:** replace each `panic!` with `tracing::warn!` + skip the
  malformed line; transcripts are already forward-compatible per
  the `TranscriptLine` enum, so unknown shapes should be tolerated.

### C2. 11 panics in LLM stream parsers
- `crates/llm/src/stream.rs:342,378,439,470,1084,1106,1120,1144,1175,1189,1208`
- `panic!("expected text")` / `panic!("expected tool calls")` after
  matching on enum variants from upstream SSE chunks. A malformed
  payload from any provider crashes the entire LLM runtime. Three
  providers, multiple pathways — high exposure.
- **Fix:** convert to `Err(StreamError::ProtocolMismatch)` and let
  the retry/circuit-breaker layer handle it.

### C3. TaskFlow tool wiring registers per-agent but misses runtime
      ctx propagation
- `src/main.rs` (registration around line 1196) — verify
- The `TaskFlowTool` is registered against a shared `FlowManager` but
  `AgentContext.session_id` is the source of `owner_session_key`. If
  the runtime constructs ctx without `with_session_id` for any code
  path (heartbeat tick, delegation receive), `taskflow` calls error
  with "requires a session context". Audit confirms the runtime sets
  `session_id` on per-session ctx but not on heartbeat/delegation
  paths.
- **Fix:** synthesise a deterministic session_id (`Uuid::nil()` or
  agent-scoped) for non-session paths, or document that taskflow is
  session-only and reject heartbeat-triggered calls early with a
  clearer error.

## 🟠 High — silent failures or wrong-by-default

### H1. Redaction false positives on `hex_token_32`
- `crates/core/src/agent/redaction.rs:103`
- `\b[a-fA-F0-9]{32,}\b` matches legitimate 32-char hex: git
  commit SHAs, S3 ETags, content hashes, MD5 artefacts. Once
  redaction is enabled, transcripts get `[REDACTED:hex_token_32]`
  on totally innocuous strings.
- **Fix:** raise the floor to 64+ chars (real secrets are typically
  longer than git SHAs which top out at 40), or document and ship
  an `exclude_patterns` config.

### H2. Audit log writes can interleave under concurrency
- `extensions/onepassword/src/audit.rs:35-68`
- POSIX `O_APPEND` is atomic per `write()` syscall, but
  `writeln!` on a `File` handle can issue two writes (data + `\n`).
  Under sustained concurrent use, partial records are possible.
  Errors are stderr-only.
- **Fix:** serialize the JSON with the trailing newline into a single
  buffer, then one `write_all`. Prefer `OpenOptions::append(true)` +
  `BufWriter` with a process-wide mutex for batched writes.

### H3. FTS rebuild is not transactional
- `crates/core/src/agent/transcripts_index.rs:149-201`
- `DELETE WHERE agent_id = ?` followed by an INSERT loop is not
  wrapped in a SQLite transaction. A crash between DELETE and INSERT
  leaves the index permanently empty for that agent until the next
  rebuild — silent partial state.
- **Fix:** wrap in `BEGIN IMMEDIATE; … COMMIT;` or use sqlx's
  transaction API. Cheap, well-understood, no UX change.

### H4. Two redaction implementations diverge
- `extensions/onepassword/src/redact.rs` vs
  `crates/core/src/agent/redaction.rs`
- The 1Password extension lives in its own workspace (no nexo-core
  dep) and ships a copy-pasted mini-redactor with 5 patterns. When
  the nexo-core list grows or a pattern's regex is fixed, this copy
  silently misses the change — secrets that get redacted in
  transcripts can still leak through 1Password tool stdout/stderr.
- **Fix:** extract the pattern table to a `redaction_patterns.toml`
  shipped with the workspace and `include_str!`-included by both
  crates, or add a doc comment + CI check that asserts the pattern
  list strings match.

### H5. Circuit breaker missing on Telegram HTTP
- `crates/plugins/telegram/src/bot.rs:81-127`
- CLAUDE.md states "circuit breaker on every external call". The
  Telegram plugin makes naked `reqwest::Client::post(...).send()`
  calls without the `nexo_resilience::CircuitBreaker` wrapper that
  every LLM client and the MCP HTTP client uses. A Telegram outage
  can cascade into thread starvation.
- **Fix:** wrap the bot's HTTP calls in the same per-channel breaker
  pattern used by the WhatsApp plugin (verify path against actual
  WA reference).

### H6. Circuit breaker missing on Google OAuth
- `crates/plugins/google/src/client.rs:158`
- Same shape as H5 — `GoogleAuthClient::http` issues raw reqwest
  calls. Token refresh under a Google outage can block the agent
  on every gmail/calendar/drive tool call until the breaker would
  have opened.
- **Fix:** wrap `token` refresh and any `request_with_retry`
  invocations.

### H7. Session_logs tool bypasses redactor + FTS index when used
- `crates/core/src/agent/session_logs_tool.rs:75`
- The tool constructs a `TranscriptWriter::new(...)` (the bare
  constructor) instead of `with_extras(redactor, index)`. The agent
  itself uses `with_extras` via `llm_behavior.rs`, so transcripts
  written through the LLM loop are redacted/indexed but anything
  the agent writes via the `session_logs` tool is not.
- **Fix:** plumb `ctx.redactor` and `ctx.transcripts_index` into the
  tool's `call` handler and use `with_extras`.

### H8. `inject_template` not declared in `plugin.toml`
- `extensions/onepassword/plugin.toml:8` vs
  `extensions/onepassword/src/tools.rs:125`
- The new tool is registered in `dispatch` and `tool_schemas()` but
  the manifest's `tools = [...]` list still has the old set. Tool
  discovery via manifest may not surface it; depends on whether
  the runtime trusts manifest or handshake.
- **Fix:** add `"inject_template"` to the manifest list.

### H9. Concurrent appends to same session can write duplicate
       headers
- `crates/core/src/agent/transcripts.rs:105-170`
- The "if file does not exist, write header first" check-then-write
  is not atomic. Two simultaneous first appends both see
  `exists=false`, both write the header. Subsequent reads parse two
  `Session` lines for the same session — `read_session` then panics
  per C1.
- **Fix:** use a per-session async mutex (DashMap-backed) around
  `append_entry`, or atomic file lock (`fs2::FileExt::try_lock`).

### H10. Watchdog kills downstream but `op inject` may still block
- `extensions/onepassword/src/op_cli.rs:281-343` (run_inject_with_command)
- The pattern spawns `op inject`, captures full stdout, then spawns
  the downstream command with that stdout as stdin. If the
  downstream command exits early with broken pipe, `op inject` got
  SIGPIPE and is fine; but the watchdog only kills the downstream
  child, not `op inject`. If `op inject` is the slow one, it keeps
  running past the timeout.
- **Fix:** put both children behind the same watchdog or use a
  process group / `setsid` and kill the group on timeout.

### H11. Plugin tool descriptions bake env-var names into
        text — operator sees them, LLM also sees them
- multiple extensions
- Tool descriptions like `"Requires CLOUDFLARE_ALLOW_PURGE=true."`
  are injected into the LLM prompt. Useful for hint, but also leaks
  the operator's enable-mechanism shape and tells the LLM exactly
  which env to ask the user to set. Mild prompt-injection surface.
- **Fix:** keep the hint in the error response, not the tool
  description; or move the description to a side-channel the LLM
  reads only on failure.

## 🟡 Medium — wrong-but-bounded, or hidden-by-default

### M1. Skill bin_versions: user-supplied regex without size limit
- `crates/core/src/agent/skills.rs:524-535` (invoke_and_parse)
- Skill author can ship a catastrophic-backtracking regex
  (`(a+)+b`). The 5-second timeout caps the damage but every probe
  burns CPU.
- **Fix:** `RegexBuilder::size_limit(64*1024).build(...)` or
  validate at frontmatter parse.

### M2. FTS5 `:` not escaped in phrase mode
- `crates/core/src/agent/transcripts_index.rs:218-231`
- `escape_fts_phrase` doubles `"` only. FTS5 phrase mode treats
  most operators as literals when wrapped in quotes, but `:` can
  still hit field-prefix syntax in some pathological inputs. Test
  doesn't cover `field:value` strings.
- **Fix:** add a test with `field:value` and `OR NOT AND` strings to
  prove the escape holds. If it doesn't, blacklist or replace.

### M3. SQL "schema migration" via `ALTER TABLE … .ok()`
- `crates/memory/src/long_term.rs:162-165`
- ADD COLUMN errors are swallowed for idempotency. Disk-full or
  permission errors during a real migration also get swallowed,
  leaving the schema half-migrated until the next start.
- **Fix:** check the error kind: only swallow "duplicate column"
  variants; surface anything else.

### M4. Help text incomplete
- `src/main.rs:2408-2430`
- 14+ Mode variants have no help line: `SetupInteractive`,
  `SetupList`, `SetupDoctor`, `SetupTelegramLink`, `CheckConfig`,
  `Reload`, the entire `Pollers*` family, `Admin`, `McpServer`.
- **Fix:** one line per variant. Mechanical.

### M5. Test ignored — policy decision pending
- `crates/core/src/agent/skills.rs:745-766`
- `requires_env_records_missing_vars_warn_only` is `#[ignore]`d with
  a TODO. After the strict/warn/disable refactor this session, the
  test's intent is now achievable — `mode: warn` is the policy this
  test wanted.
- **Fix:** rewrite the test to set `requires.mode: warn` in the
  frontmatter and assert the skill loads with the banner. Drop
  `#[ignore]`.

### M6. `let _ = hooks.fire(...).await` and similar in llm_behavior
- `crates/core/src/agent/llm_behavior.rs:640,660,664,763,795`
- Errors from extension hooks and memory persistence are silently
  dropped. The agent's audit trail (and any hook-based side effect)
  fails without log.
- **Fix:** at minimum, `if let Err(e) = … { tracing::warn!(error =
  %e, "hook failed"); }`.

### M7. Anthropic error body discarded
- `crates/llm/src/anthropic.rs:113,117,136`
- `response.text().await.unwrap_or_default()` in error paths drops
  the response body if it's not valid UTF-8 or if the body read
  errors. Means real provider errors come through as empty strings.
- **Fix:** read as `bytes()`, then `String::from_utf8_lossy` so we
  always have *something* to log.

### M8. Tokio Mutex held across .await in config_reload
- `crates/core/src/config_reload.rs:118,133,158,171,353`
- `let _gate = self.gate.lock().await; … .await; …` — sequential
  awaits while holding the gate. Currently not deadlock-prone (no
  reverse lock acquisition seen) but fragile.
- **Fix:** drop the guard before the inner await, or refactor to
  hold `RwLock::write` only for the critical region.

### M9. `ssh_exec_writes` documented as enabling `scp_upload` only —
       but the toggle currently gates more
- `extensions/ssh-exec/src/tools.rs`
- Capabilities inventory entry says `SSH_EXEC_ALLOW_WRITES` →
  "Allow `scp_upload`". Verify in source whether this also gates
  `ssh_run`'s ability to mutate remote state via shell.
- **Fix:** sync the description in
  `crates/setup/src/capabilities.rs::INVENTORY` to match what the
  toggle actually unlocks.

### M10. Order-dependent redaction can produce nested markers
- `crates/core/src/agent/redaction.rs:68-80`
- Patterns run in declared order over already-replaced text. If a
  later pattern matches the literal `REDACTED`, you get
  `[REDACTED:[REDACTED:…]]`. Unlikely with current patterns; future
  custom patterns can hit it.
- **Fix:** collect all match spans first, then apply replacements
  right-to-left. Or document the ordering contract.

### M11. Default version regex rejects `4` (major-only)
- `crates/core/src/agent/skills.rs:81`
- `\d+\.\d+(?:\.\d+)?` requires at least major.minor.
  `normalize_version_string` later pads, but only for matched
  output. A bin emitting "v4" is treated as ParseFailed.
- **Fix:** relax to `\d+(?:\.\d+){0,2}` or document the lower bound.

### M12. `runtime_snapshot.rs` exposes private type
- `crates/poller-ext/src/lib.rs:89` (warning seen during build)
- `pub fn ExtensionPoller::with_tools_cache` exposes
  `ToolDefinition` which is `pub(crate)`. Compiles with warning
  but breaks downstream consumers who try to name the type.
- **Fix:** make `ToolDefinition` `pub` or change the method to
  accept `impl Into<…>` / a generic.

## 🟢 Low — cosmetics, dead config, hardening polish

### L1. `std::env::set_var` in tests without a process-wide guard
- `crates/setup/src/capabilities.rs::tests` uses a local
  `Mutex<()>` per file. `crates/core/src/agent/redaction.rs::tests`
  doesn't (passes today, fragile if more parallel tests touch the
  same vars).
- **Fix:** standardize on `serial_test::serial` for any test that
  mutates env.

### L2. `let _ = std::fs::set_permissions(...)` swallows perm errors
- `crates/auth/src/gauntlet.rs:50`
- If chmod 0o600/0o700 fails on a credential file, secrets remain
  world-readable and the agent doesn't log a thing.
- **Fix:** log + return error.

### L3. SSE parser background tasks never get a shutdown signal
- `crates/llm/src/stream.rs:679,869,1007`
- Tasks spawned for OpenAI / Anthropic / Gemini SSE parsing keep
  reading until EOF; channel drop on receiver-side stops the parser
  via send-error, but a misbehaving server that never closes leaves
  the task pinned.
- **Fix:** thread the runtime cancellation token into each parser.

### L4. `let _ = notify_tx.try_send(())` drops watcher pings
- `crates/core/src/config_watch.rs:59`
- Channel-full signals are dropped silently. Under load the agent
  may miss a config-reload trigger.
- **Fix:** use `mpsc::channel` with a cap of 1 and `try_send` →
  ignore — that's actually what the current code does on purpose
  (coalesce), but a comment would help future readers.

### L5. `commits.last().unwrap()` on memory_history empty repo
- `crates/core/src/agent/memory_history_tool.rs:54`
- Empty workspace_git → `git.log(limit)` returns `Vec::new()` →
  `.last().unwrap()` panics.
- **Fix:** `.last().ok_or_else(|| anyhow!("no commits yet"))?`.

### L6. `expect("write media bytes")` panics on disk full
- `crates/core/src/agent/llm_behavior.rs:962`
- Persisting media attachment uses `expect`. Disk full crashes the
  agent on the next user-attached image.
- **Fix:** propagate the error to the LLM as a tool error.

### L7. `expect("failed to build reqwest client")` at startup
- `crates/llm/src/minimax.rs:90-93`
- A reqwest client build failing at startup is exotic but the
  `expect` panics instead of returning structured error.
- **Fix:** propagate to `LlmRegistry::build` so the agent fails
  with a config error instead of panicking.

### L8. `std::env::var("MINIMAX_GROUP_ID")` falls back silently
- `crates/llm/src/minimax.rs:452,455`
- Two `.ok()` paths: env var missing → file missing → `None`. If
  the operator created the file with a typo (extra newline, wrong
  group), the file content silently overrides env. Hard to debug.
- **Fix:** log at debug which source resolved the group_id.

### L9. Capabilities inventory hardcoded; no compile-time guard
      against adding a new env toggle without updating it
- `crates/setup/src/capabilities.rs::INVENTORY`
- Future extension authors will forget to update the inventory.
- **Fix:** add a CLAUDE.md note (or a CI grep) that fires on any
  `_ALLOW_` env var introduced under `extensions/`. Cheap.

### L10. `archive/` directory pre-existing in working tree
- gitignored or stray? `git status` shows `??`.
- Not a code bug; cleanup item.
