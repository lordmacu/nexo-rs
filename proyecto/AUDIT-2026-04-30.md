## Critical findings (fix before next phase)

### C1 — `EffectiveBindingPolicy` missing fields for half the new policies
**Severity: HIGH.** `crates/core/src/agent/effective.rs:45-93` resolves `remote_triggers` + `proactive` per-binding, but **has no fields for `lsp`, `config_tool`, `team`, `repl`**. Per-binding override silently ignored on hot-reload — only agent-level YAML hits.
**Fix**: extend struct + resolution fns + `binding_validate` + `runtime_snapshot::build`. Mirrors the existing `proactive` resolution.

### C2 — Tool registration is one-shot at boot
**Severity: HIGH.** Every Phase 79 tool registered inside per-agent boot loop (`src/main.rs:2042-2705`). No reload path re-runs registration. Toggling `lsp.enabled`, `team.enabled`, `proactive.enabled`, `repl.enabled`, `config_tool.self_edit`, growing `remote_triggers` → daemon restart required. Phase 18 docs imply otherwise.
Only ONE post-hook registered: `main.rs:3492` (`PairingGate::flush_cache` from Phase 70.7). Need analogues for cron-store / REPL registry / LSP manager / team router cache / tool registry per-agent.
**Fix**: `ConfigReloadCoordinator::register_post_hook` for each subsystem; wrap registration in a `rebuild_per_agent_tools()` callable from reload.

### C3 — `capabilities.rs::INVENTORY` not updated for any Phase 77/79 toggle
**Severity: HIGH (CLAUDE.md mandate violation).** `crates/setup/src/capabilities.rs:90-171` ends at `EMAIL_INSECURE_TLS`. Missing: `NEXO_CLAUDE_CLI_VERSION` (anthropic_auth.rs:63), `config-self-edit` Cargo feature surface, any `NEXO_REPL_*` / `NEXO_LSP_*` knob, `NEXO_PROACTIVE_*`. `agent doctor capabilities` silently incomplete.
**Fix**: append entries; same commit as the toggles per CLAUDE.md rule.

### C4 — 77.9 + 77.10 + 77.11 modules orphaned
**Severity: HIGH (security-relevant + UX).**
- `crates/driver-permission/src/{sed_validator,path_extractor}.rs` (1 261 LOC + 33 tests) — only `bash_destructive::check_sed_in_place` reaches the decider (`mcp.rs:200`); the richer allowlist/denylist is dead.
- `crates/driver-permission/src/should_use_sandbox.rs` (401 LOC + 20 tests) — zero callers outside its own tests; sandbox decision computed and discarded.
- `crates/llm/src/rate_limit_info.rs` (762 LOC) — builds plan-hint; never propagated. No `LlmError::QuotaExceeded { retry_after, plan_hint }` variant. `setup doctor` doesn't read recent telemetry.
**Fix**: wire validators into `decider.rs`; add `LlmError::QuotaExceeded`; expose in `setup doctor`.

### C5 — `SecretGuardConfig` YAML not read
**Severity: MEDIUM.** `src/main.rs:802` and `:8632` carry `// TODO: read from memory.secret_guard YAML when config dep allows.` Both unconditionally build `SecretGuardConfig::default()`. Operator can't toggle `Block`/`Redact`/`Warn` or exclude rules. Whole `secret_config.rs:79-130` schema dead.
**Fix**: add to `crates/config` types, plumb through `RuntimeConfig` → main.rs.

## Major findings

### M1 — `tools/list_changed` notification advertised disabled
`crates/core/src/agent/mcp_server_bridge/bridge.rs:253-258` declares `"tools": { "listChanged": false }`. Even if hot-reload wired the call (it doesn't — Phase 79.M.h tracked open in FOLLOWUPS), clients ignore it. Net: hot-reloading `mcp_server.expose_tools` requires process restart.

### M2 — MCP audit `args_size_bytes` + `args_hash` always 0/None
`crates/mcp/src/server/dispatch.rs:706-707` hard-codes both fields; `audit_log/sqlite_store.rs:71` schema column always 0. `SELECT … args_size_bytes FROM mcp_call_audit` analytics misleading.
**Fix**: hash params with sha256 + record `serde_json::to_vec(&args).len()`.

### M3 — Coordinator vs proactive mutual exclusion not enforced
`crates/core/src/agent/binding_validate.rs:400` accepts both flags. Spec mandated incompatibility (coordinator delegates → shouldn't tick).
**Fix**: 1-line rejection with clear error.

### M4 — `extractMemories` + `autoCompact` only run inside `driver-loop`
Both wired in `crates/driver-loop/src/{extract_memories,orchestrator}.rs`. Agents not running through driver-loop (regular `AgentRuntime`) never get them.
**Fix**: factor a shared service or boot in both paths.

### M5 — `cron_tool_bindings` frozen at boot
`main.rs:3052-3128` captures `Arc::clone(&effective)` once. Reload changing `allowed_tools` / `dispatch_policy` for an agent → cron firings use OLD policy.
**Fix**: post-hook flush analogous to PairingGate.

### M6 — `PostCompactCleanup` is a stub + `CompactSummaryStore::forget()` no-op
`post_compact_cleanup.rs:38-48` only ticks extract counter. Leak's `postCompactCleanup.ts` resets MicroCompactState turn counter, surfaced-memory caches, `compactWarningState`. `compact_store.rs:68-74` `forget()` left as TODO.
**Fix**: low-effort port; mirror leak.

### M7 — REPL semantically diverges from leak
Leak `primitiveTools.ts:21-39` makes REPL a VM hosting FileRead/Write/Edit/Glob/Grep/Bash/NotebookEdit/Agent. Ours is subprocess pool for python/node/bash (`repl_registry.rs:59-90`). No sandbox, no isolation, no nsjail/firejail/bwrap. `repl_tool.rs` itself has zero tests.
**Decision**: re-spec as our own thing OR port the VM model. Don't claim leak parity.

### M8 — 79.2 deferred-schema only used by MCP catalog
`tool_search_tool.rs:30-37` itself flags this. Leak defers TodoWrite, LSP, TeamCreate, RemoteTrigger, NotebookEdit, REPL by default. Infrastructure tested but token-budget win is partial.
**Fix**: mark `ToolMeta::deferred()` on the 6 tools above.

### M9 — `expose_tools` typo path silent
`src/main.rs:9263-9266` warns on unknown name. No test. Future tool rename → silent drop.
**Fix**: catalog test enumerating every canonical name.

### M10 — `MUTATING_TOOLS` lists `TeamCreate`/`TeamDelete` twice
`crates/core/src/plan_mode.rs:295,316` and `:314,315`. Harmless dedup; code smell.
